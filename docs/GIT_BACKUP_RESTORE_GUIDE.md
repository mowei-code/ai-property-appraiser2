# Git 備份與還原操作指南 (Git Backup & Restore Guide)

本文件旨在提供穩定、安全的專案備份與還原教學，確保在系統不穩定時能迅速回歸正常狀態。

---

## 🛡️ 安全第一：操作前的黃金法則
在執行任何「還原」指令（如 `reset` 或 `restore`）之前，請務必先執行**手動實體備份**：
*   **做法**：直接複製整個專案資料夾至其他位置（如桌面或外部雲端硬碟）。
*   這是唯一一種不依賴任何軟體、絕對不會出錯的方法。

---

## 📦 備份 (Backup) 的三種等級

### 1. 輕量級：本地存檔 (Commit)
每次完成一個功能或修復一個 Bug 時執行：
```bash
git add .
git commit -m "描述您的變更內容"
```

### 2. 里程碑：版本標籤 (Tag) - **推薦**
當專案處於完美、穩定的狀態時，為其取一個名字：
```bash
git tag -a stable-v1 -m "功能完備的穩定版本"
git push origin stable-v1
```

### 3. 保險箱：雲端同步 (Push)
將本地所有紀錄同步到 GitHub：
```bash
git push
```

## ⏪ 還原 (Restore) 的三種情境

### 情境 A：剛改壞了，想撤銷「尚未存檔」的變更
如果您只是剛剛動手改了幾行，程式碼就亂掉了：
*   **安全做法（暫存）**：
    ```bash
    git stash
    ```
    *這會把目前的變更收起來。如果發現收錯了，輸入 `git stash pop` 即可拿回來。*

*   **徹底清除（放棄）**：
    ```bash
    git reset --hard HEAD
    ```
    *注意：這會直接刪除所有未存檔的內容，無法找回！*

### 情境 B：想回到特定的「穩定標籤」(如 stable-v1)
當您做了好幾次 commit，但發現還是現在這個標籤最穩定：
```bash
git checkout stable-v1
```

### 情境 C：本地完全亂掉，想抓 GitHub 網頁版救命
1.  到 GitHub 專案網頁點擊 **[Code]** -> **[Download ZIP]**。
2.  解壓縮後，您就有一份百分之百乾淨、不帶任何指令風險的原始碼。
3.  或者使用指令強制覆蓋本地：
    ```bash
    git fetch origin
    git reset --hard origin/main
    ```

## 📌 常見問題小秘訣
*   **看歷史紀錄**：輸入 `git log --oneline` 可以查看過往所有的存檔點與編號。
*   **紅字錯誤 (tsconfig)**：如果看到配置檔案紅字，請確認是否包含目前的原始碼路徑。
*   **排除檔案**：不想傳上雲端的檔案請丟進 `backups/` 資料夾，我已設定好自動忽略。
